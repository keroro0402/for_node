モジュールについて

モジュールとは「ソース」コードを機能ごとに細かく分解した単位
なぜ、モジュールが必要か？？

モジュール機能を使わないと一つのファイルに複数の機能をプログラミングすることになる。
つまりコードが助長になり見通しが悪く、メンテナンス性が大きく低下するのである。
それを「モジュール」機能を使うと、一つの機能に一つのプログラミングが出来るようになるのである。

モジュール管理システムについて
モジュール管理システムは「２つ」の管理システムがある。
「ESM」 と[CJS]である

【CJS】とは
・ブラウザでは使えない。Node.jsでのみ使える。
・モジュールを読み込むときは「require」で行う
・モジュール側にはmodule.exports = モジュール名（関数名）でexportする

【ESM】とは
.ブラウザで使える。Node.jsでも使える。
.モジュールを読み込むときは「import/export」
.Node.jsで使うときは明示的に指定しないと使えない

【ESMの時の手順】
1:package.jsonにtypeプロパティを設定して、バリューにmoduleをする
{
    "type" : "module"
}
2:モジュール側はexportで出力させる
export {} 
3:読み込む側ではimportで読み込む
import {} from "モジュール名"
